Operating Instructions

How to Use This Timetable

01
The Golden Rules
  • The 20–30 min breaks after every 2-hour block are not optional — they are part of the system. Your brain consolidates what it just learned during rest. Skipping them costs more time than they save.
  • Never skip two days of the same subject back to back. Consistency beats intensity every time.
  • The school/coursework hour every weekday means assignments never pile up into a crisis. Stay ahead of the syllabus — do not let academic work compete with your personal growth.
  • The 6:00–7:00 morning block on free days is your edge. Most people are still asleep. Use it without exception.
  • Tech reading is not passive — take notes. One article a day from OWASP Blog, AWS What's New, or The Pragmatic Engineer minimum.
02
Sunday Review — Every Week
  • Sunday is your heaviest independent study day — treat it like a second Tuesday, not a rest day. The rest is built into the schedule already.
  • What did you complete this week versus what you planned?
  • Which subject got skipped the most — and why?
  • What is one thing you built or wrote this week?
  • What do you want to finish by this time next week? Set it in the tracker before you sleep.

The Stack

What You're Learning and Why

React — FreeCodeCamp

You already know vanilla JS and you have built real projects. React is the next layer. Do not just watch — build alongside the video. Every concept you watch should be coded by hand in a separate file before you move on.

Python — Bro Code

Python is the language of AI/ML, automation, and backend scripting. You are 2 hours in. Keep the momentum. When you finish Bro Code, start building small tools — a file organiser, a web scraper, a simple API. That is what makes it stick.

OWASP / Secure Coding

Start with the OWASP Top 10 at owasp.org — it is free. Read one vulnerability per session: what it is, how it is exploited, and how to prevent it. Then go into your own web projects and find where you have the same vulnerability. This is how developers who think like security engineers are built.

AWS Cloud Fundamentals

Start with AWS Cloud Practitioner Essentials on AWS Skill Builder — it is free. Understand the concepts first: what is a VPC, what is IAM, what is S3, how does EC2 work. The certification follows naturally after the understanding.

Projects / Build Time

This is where learning becomes skill. Do not let project time become tutorial time. Build things. Break things. Fix them. Your portfolio is what gets you in rooms before your degree does. Luna Commerce is already proof of that — keep building at that level.

School / University Coursework

One dedicated hour per weekday — lecture notes, assignments, past questions, course reading. This keeps academic pressure from ambushing you before exams. Done consistently, it never becomes a crisis. Done inconsistently, it becomes the reason everything else falls apart.

"The people at the top did not get there by waiting for the right time. They got there by starting before they were ready and staying consistent when it stopped being exciting."
